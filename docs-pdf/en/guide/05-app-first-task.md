# Complete your first task with Codex

::: tip final check
The last date of verification of official information: 2026-05-27. This chapter takes developing a simple web page as an example to demonstrate the complete operation process from creating a working directory to task iteration.
:::

This chapter takes "Using Codex Desktop App to develop a simple web page about the history of AI development" as an example to complete a complete closed loop of the task and help you quickly get started with the basic usage of Codex Desktop App.

## Step 1: Create a local working folder

Create an empty folder locally as the working directory of Codex. All files generated by Codex will be saved here.

::: warning
Try not to include Chinese characters in the folder path to avoid compatibility issues with some tools.
:::

![Create a new local folder](../images/image-20260511175758960.png)

## Step Two: Choose Conversation or Project

After opening the Codex Desktop App, you need to choose between "Conversation" or "Project" to start the task.

- **Conversation**: Suitable for one-time tasks, simple operation, but the working directory is not shared between multiple conversations.
- **Project**: Supports the creation of multiple conversations in the same working directory. Each conversation handles different subtasks, making management more convenient.

If you're not sure which one to choose, **choose the project first** - it's more flexible for subsequent expansion tasks, and the working directory only needs to be configured once.

## Step 3: Add working directory

After selecting the project, click "Use existing folder" and select the folder you just created.

![Select working directory](../images/image-20260511180342249.png)

After the selection is completed, the path of the current working directory will be displayed in the lower left corner of the dialog box. Just confirm that the path is correct.

![Confirm working directory](../images/image-20260511180544468.png)

## Step 4: Enter the task description and start execution

Enter your requirements in the dialog box, click Send, and Codex will start executing the task.

![Enter task and send](../images/image-20260511180648409.png)

After the task is completed, Codex will display the results in the dialog and write the generated files to the working directory.

![Mission accomplished](../images/image-20260511184050002.png)

If a web page file is generated, you can directly click the "Open" button that pops up in Codex to preview the effect in the app's built-in browser without manually opening the folder.

![Built-in browser preview](../images/image-20260511184146656.png)

## Step 5: Iterate step by step

If you are not satisfied with the result, you can continue to describe the modification requirements directly in the current dialog box, and the Codex will make adjustments based on the existing results.

As the dialogue turns increase, the context window gradually fills up. Click the small circle icon in the lower right corner of the dialog box to view the current context usage.

![Contextual usage](../images/image-20260511184315443.png)

::: info What is a context window?
Each conversation has a capacity limit, and each Q&A round takes up a certain amount of space. When the context is nearly full, it is recommended to create a new conversation in the project to continue processing subsequent tasks - the new conversation still shares the same working directory and does not require reconfiguration.
:::

![Create a new conversation in the project](../images/image-20260511184528218.png)

If you use the "conversation" mode instead of a project, you need to re-specify the working directory when creating a new conversation. The two conversations are also isolated from each other, making it inconvenient for unified management. This is one of the main reasons why projects are recommended.
