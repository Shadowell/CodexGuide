# Building a roadmap together

Goal: Make this repository a Codex practical knowledge base for beginners, creators, developers and teams around the world.

## The first stage: complete skeleton

- [x] Create VuePress + Theme Hope documentation station.
- [x] Establish a README, contribution guidelines, code of conduct, and security policies.
- [x] Build entry-level, advanced, team and case directories.
- [x] Add official information index.
- [ ] Added first batch of screenshots and command output.
- [ ] Add "Applicable version" and "Last verification date" to each tutorial.

## Phase Two: Guideline Shaping

- [ ] Complete tutorial on CLI from installation to daily use.
- [ ] Complete the `AGENTS.md` topic.
- [ ] Complete the prompt word template library.
- [ ] Complete the first edition of the Troubleshooting Manual.
- [ ] Complete the usage differences between Cloud, IDE, and Desktop App.

## The third stage: case driven

- [ ] Collect 20 real engineering cases.
- [ ] Each case includes task description, key processes, verification methods and review.
- [ ] Create case tags: front-end, back-end, testing, refactoring, documentation, CI, security.
- [ ] Added failure cases to illustrate scenarios where Codex is not suitable for direct processing.

## Phase 4: Community spread

- [ ] Design a more complete homepage and Open Graph.
- [ ] Create a Chinese and English keyword index.
- [ ] Publish GitHub Pages.
- [ ] Organize short links and learning roadmap suitable for social media communication.
- [ ] Invite early contributors to claim chapters.

## Contribution entrance

You can start with these tasks:

- Correct inaccuracies in documentation.
- Add a new Codex case that you have personally run.
- Fix a common error in the Troubleshooting Guide.
- Synchronize new changes in official documents to the data index.
